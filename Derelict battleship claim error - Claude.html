The Core Problem

Your ship claiming logic assumes that ships.number is a unique identifier that players reference, but ship ID 11 has number = 0. This creates several issues:

  1. Multiple ships could have number = 0 (or other duplicate numbers)
  2. Players with ship = 0 might mean "no ship" rather than "ship with number 0"
  3. The relationship is ambiguous

Quick Fix Options

Change the players table to reference ships.id instead of ships.number:

    SELECT s.id FROM ships s WHERE s.number = players.ship

c
static const char *SQL_CHECK =
